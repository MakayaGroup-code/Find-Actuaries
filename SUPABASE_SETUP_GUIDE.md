# Find Actuaries – Supabase Database Setup Guide

This guide will connect your Find Actuaries platform to a real backend using **Supabase**.

## Step 1: Create a Supabase Project

1. Go to [https://supabase.com](https://supabase.com)
2. Sign up / Log in
3. Click **"New Project"**
4. Fill in:
   - **Name**: `find-actuaries`
   - **Database Password**: Create a strong password (save it!)
   - **Region**: Choose closest to your users (e.g. Europe West or Singapore)
5. Click **Create new project** (takes ~2 minutes)

## Step 2: Get Your API Keys

1. In your Supabase dashboard, go to **Project Settings** → **API**
2. Copy these two values:
   - `Project URL` → `NEXT_PUBLIC_SUPABASE_URL`
   - `anon public` key → `NEXT_PUBLIC_SUPABASE_ANON_KEY`

## Step 3: Add Environment Variables

Create a file called `.env.local` in the root of your project:

```env
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key-here
```

**Never commit this file to Git.**

## Step 4: Run the Database Schema

Go to **SQL Editor** in Supabase and run the following SQL:

```sql
-- Enable UUID extension
create extension if not exists "uuid-ossp";

-- PROFILES TABLE (Actuaries)
create table if not exists profiles (
  id uuid references auth.users on delete cascade primary key,
  full_name text,
  title text,
  company text,
  location text,
  experience_years integer default 0,
  bio text,
  specialisms text[] default '{}',
  availability text default 'Open to opportunities',
  linkedin text,
  created_at timestamp with time zone default timezone('utc'::text, now())
);

-- JOBS TABLE
create table if not exists jobs (
  id bigint generated by default as identity primary key,
  title text not null,
  company text not null,
  location text,
  type text,
  salary text,
  experience text,
  description text,
  posted_by uuid references auth.users,
  created_at timestamp with time zone default timezone('utc'::text, now())
);

-- APPLICATIONS TABLE
create table if not exists applications (
  id bigint generated by default as identity primary key,
  job_id bigint references jobs on delete cascade,
  applicant_id uuid references auth.users on delete cascade,
  status text default 'Applied',
  created_at timestamp with time zone default timezone('utc'::text, now()),
  unique(job_id, applicant_id)
);

-- Enable Row Level Security
alter table profiles enable row level security;
alter table jobs enable row level security;
alter table applications enable row level security;

-- Policies (Basic - can be tightened later)
create policy "Public profiles are viewable by everyone"
  on profiles for select using (true);

create policy "Users can insert their own profile"
  on profiles for insert with check (auth.uid() = id);

create policy "Users can update own profile"
  on profiles for update using (auth.uid() = id);

create policy "Jobs are viewable by everyone"
  on jobs for select using (true);

create policy "Authenticated users can post jobs"
  on jobs for insert with check (auth.role() = 'authenticated');

create policy "Users can view their own applications"
  on applications for select using (auth.uid() = applicant_id);

create policy "Users can apply to jobs"
  on applications for insert with check (auth.uid() = applicant_id);
```

## Step 5: Enable Authentication

1. Go to **Authentication** → **Providers**
2. Enable **Email** provider
3. (Optional but recommended) Enable **Google** and **LinkedIn** providers

## Step 6: Update Your Code

The project already includes `lib/supabase.ts`.

To use it in any page/component:

```tsx
import { supabase } from '@/lib/supabase'

// Example: Fetch jobs
const { data: jobs } = await supabase.from('jobs').select('*')
```
